# Focus Bear Onboarding Experience Review

## Date and Time of Testing
I completed the Focus Bear onboarding process on **16 March 2026 at approximately 4:00 PM (Melbourne time).**

During the onboarding process, I explored the main setup steps and tested different features as a first-time user. Below are several usability issues I encountered.

---

# Issue 1 – "Adjust with AI" Feature Caused App Instability

(screenshots/issue_1.jpeg)

### Problem
While using the “Adjust with AI” feature, the application became unstable and behaved unexpectedly. In some cases the page stopped responding or did not update properly after the adjustment.

### Why This Is a Problem
New users may lose confidence in the feature if the system behaves unpredictably during onboarding.

### Suggested Improvement
Ensure the AI adjustment process runs more reliably and provide loading indicators or confirmation messages so users understand what the system is doing.

---

# Issue 2 – Page Refresh Causes Habit Data Loss

(screenshots/issue_2.jpeg)

### Problem
When editing a habit during onboarding, refreshing the page causes all entered information to disappear.

### Why This Is a Problem
Users may accidentally refresh the page or navigate away, which results in losing all progress. This can be frustrating during onboarding.

### Suggested Improvement
Possible solutions:
- Auto-save habit inputs
- Show a warning message before refreshing
- Preserve form data in the session

---

# Issue 3 – Missing Navigation / Back Button

(screenshots/issue_3.jpeg)

### Problem
Some onboarding screens do not include a clear navigation bar or back button.

### Why This Is a Problem
If users want to review a previous step or correct something, they may not know how to return to the previous page.

### Suggested Improvement
Add a visible Back button or navigation bar so users can easily move between steps during onboarding.

---

# Issue 4 – Error Messages Are Difficult to Understand

(screenshots/issue_4.jpeg)

### Problem
Some error messages appear technical or unclear and may not be easy for users to understand.

### Why This Is a Problem
Users with ADHD or executive functioning challenges may feel confused if the error message does not clearly explain what went wrong or how to fix it.

### Suggested Improvement
Improve error messages by:
- Using simpler language
- Explaining the problem clearly
- Suggesting how the user can fix it

Example:

Instead of:  
"Error: Empty content"

Use:  
"Something went wrong with this step. Please check your settings and try again."

---

# Issue 5 – Web Flow Order Could Be Improved

(screenshots/issue_5.jpeg)

### Problem
The onboarding flow includes a step for **choosing app blocking** before clearly defining **frictions**.

### Why This Is a Problem
From a user perspective, it may make more sense to first define the user's **frictions or distractions**, and then choose the apps that should be blocked.

### Suggested Improvement
Reorder the onboarding flow:

1. Identify distractions or frictions  
2. Select apps or websites to block  
3. Configure focus sessions

This would create a more logical and user-centered onboarding experience.
